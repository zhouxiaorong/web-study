direction改变水平流向direction: ltr（默认值） | rtl | inherit;

注意点

  1. ltr: 文本方向从左到右, left to right
  2. rtl: 文本方向从右到左, right to left
  3. inherit: 从父类 direction 属性继承
  4. 在不支持 text-align:start/end 的浏览器中（如 IE），不同的 direction 属性值会改变 text-align 属性的初始值：当 direction 值为 ltr 的时候，text-align 的初始值是 left；当 direction 值为 rtl 的时候，text-align 的初始值是 right。

unicode-dibi设置文本重写unicode-bidi: normal | embed | bidi-override | initial | inherit;

注意点

  1. normal: 默认。不使用附加的嵌入层面。正常排列。（假设设置了 direction:rtl，则图片、按钮以及问号、加号之类的字符会从右往左显示，但是中文、英文字符还是从左往右显示。）
  2. embed: 创建一个附加的嵌入层面。独立内嵌，不受外部影响；要想起作用，只能作用在内联元素上（在通常情况下，embed属性值的表现和 normal 是一样的）。
  3. bidi-override: 创建一个附加的嵌入层面。重新排序取决于 direction 属性。
  4. unicode-bidi 属性与 direction 属性一起使用，来设置或返回文本是否被重写，以便在同一文档中支持多种语言。

white-space控制空白字符的显示，是否自动换行white-space:normal | nowrap | pre | pre-wrap | pre-line

注意点

  1. normal: 不保留换行符，空格合并成一个，会自动换行
  2. nowrap: 强制不换行，空格合并成一个（在同一行内显示所有文本，直到文本结束或者遇到
    标签）
  3. pre: 保留换行符、空格，不会自动换行，preserve的缩写。（行为方式类似 HTML 中的 pre 标签）
  4. pre-wrap: 保留换行符、空格，行前的空格超出一行时会换行，会自动换行，preserve + wrap
  5. pre-line: 保留换行符，空格合并成一个，会自动换行，preserve + new line + wrap

word-wrap控制长度超过一行的单词是否被拆分换行word-wrap：normal | break-word

注意点

  1. 是word-break的补充
  2. normal: 只在允许的断字点换行（浏览器保持默认处理）。允许内容顶开或溢出指定的容器边界。
  3. break-word: 内容在一行显示不下时，拆分换行该内容（只有长到溢出的单词才会被强制拆分换行！）
  4. word-wrap 属性原本属于微软的一个私有属性，在 CSS3 现在的文本规范草案中已经被重名为 overflow-wrap。word-wrap 现在被当作 overflow-wrap 的 “别名”。 稳定的谷歌 Chrome 和 Opera 浏览器版本支持这种新语法。

word-break控制单词如何被拆分换行word-break: normal|break-all|keep-all;

注意点

  2. normal: 使用浏览器默认的换行规则。
  3. break-all: 允许在单词内换行（所有单词碰到边界一律拆分换行）
  4. keep-all: 只能在半角空格或连字符处换行。（所有“单词”一律不拆分换行，只有空格可以触发自动换行）

设置表格的边框是分开的还是单一的 border-collapse: separate | collapse

注意点

  1. separate : 默认值。边框会被分开。不会忽略 border-spacing 和 empty-cells 属性。
  2. collapse : 边框会合并为一个单一的边框。会忽略 border-spacing 和 empty-cells 属性。

设置相邻单元格的边框间的距离（仅用于"边框分离"模式-border-collapse: separate;） border-spacing: length

注意点

  1. length:规定相邻单元的边框之间的距离。使用 px、cm 等单位。不允许使用负值
  2. 如果定义一个 length 参数，那么定义的是水平和垂直间距。
  3. 如果定义两个 length 参数，那么第一个设置水平间距，而第二个设置垂直间距。
